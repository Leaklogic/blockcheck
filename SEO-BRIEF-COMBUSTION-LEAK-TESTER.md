# SEO Brief: Combustion Leak Tester / Head Gasket Tester Site

## Objective

Build a UK-focused ecommerce site that can outrank `drheadgasket.co.uk` for high-intent combustion leak tester and head gasket tester searches by combining:

- stronger page targeting
- better UX and trust signals
- better structured data
- better internal linking
- better supporting content
- cleaner technical SEO

The goal is not to copy the current competitor's over-optimization. The goal is to keep the same relevance signals, then add stronger content, stronger trust, and stronger technical execution.

## Primary Search Terms To Target

These exact searches must be accounted for in the page strategy:

- `combustion leak test kit`
- `combustion leak tester`
- `head gaskit tester`
- `head gaskit test kit`
- `test blown head gaskit`

Important:

- Google is likely normalizing `gaskit` to `gasket`.
- Do not create spammy typo-stuffed pages.
- Use the correct spelling in core titles and headings.
- Mention common misspellings once in supporting FAQ/help copy if needed.

## On-Page Rules

### Titles

- Keep titles concise and descriptive.
- Put the primary term near the front.
- Add one secondary qualifier only if useful.
- Avoid the competitor's 150+ character title style.

Google's guidance says to:

- make sure every page has a title
- write descriptive and concise title text
- avoid unnecessarily long or verbose titles

### H1 Rules

- One clear H1 per page.
- H1 should match the page intent.
- H1 should use the main term naturally.

### Headings

Use structured subheadings such as:

- `What Is A Combustion Leak Tester?`
- `Who This Head Gasket Test Kit Is For`
- `Petrol vs Diesel Test Fluid`
- `How To Use The Tester`
- `How To Read The Result`
- `Common Testing Mistakes`
- `What Is Included`
- `Fitment and Adapter Sizes`
- `Delivery, Returns and Warranty`

## Technical SEO Requirements

- only one canonical URL per product
- no category-prefixed product duplicates
- self-referencing canonical on every indexable page
- XML sitemap for pages, categories, products, images
- noindex or block faceted junk URLs
- fast page speed
- lazy load images
- compressed WebP/AVIF where possible
- strong caching
- proper mobile layout
- clean breadcrumb HTML
- unique title and meta description on every page
- Open Graph and Twitter tags
- proper 301 redirect rules
